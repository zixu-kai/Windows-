## 使用方法

1. 解压 ZIP 文件到任意文件夹
2. **双击 `Windows组织管理修复工具.bat`**
3. 弹出 UAC 提示时点"是"
4. 按照脚本提示，每个步骤输入 `y` 确认
5. **必须重启计算机**使所有更改生效

## v1.0 覆盖的全部位置（8大来源）

| # | 来源 | 清除内容 | 出现位置 |
|---|------|---------|---------|
| 1 | 系统设置策略 | HKLM/HKCU\Software\Policies 全部子项 | 设置 → 隐私/更新/账户 各页面顶部 |
| 2 | 防火墙策略 | Policies\Microsoft\WindowsFirewall + netsh advfirewall reset | 防火墙 → "此设置由你的组织进行管理" |
| 3 | Edge 浏览器 | Policies\Microsoft\Edge + EdgeUpdate | Edge 设置 → "由组织管理" |
| 4 | Chrome 浏览器 | Policies\Google\Chrome | Chrome 设置 → "由组织管理" |
| 5 | Defender 杀毒 | DisableAntiSpyware 策略值 | Windows 安全中心 → 无法修改 |
| 6 | Explorer 限制 | NoControlPanel/DisableSettingsPage 等 | 控制面板/设置 被锁定 |
| 7 | MDM / 工作账户 | Enrollments 注册 + dmwappushservice 服务 | dsregcmd 显示 MDM 已注册 |
| 8 | 组策略缓存 | GroupPolicy 文件夹 + secedit 重置 + gpupdate | 所有 GPO 生效的策略 |

## 清除后的影响

### 无负面影响

- 删除防火墙策略：防火墙重置为默认规则，正常保护不受影响，可自由配置规则
- 删除 Edge/Chrome 策略：浏览器恢复个人模式，所有设置可自由修改
- 删除 Defender DisableAntiSpyware：杀毒功能恢复正常可用
- 删除 Explorer 限制：控制面板、设置页面恢复访问
- 删除 CloudDomainInfo：消除核心标识，不影响网络连接
- 删除 WindowsSelfHost：退出 Insider 计划，系统版本不变
- 禁用 dmwappushservice：减少遥测上报，不影响正常运行

### 需要注意

- 删除 HKLM\Software\Policies 整个目录：所有应用的组织策略消失（已自动备份）
- netsh advfirewall reset：自定义防火墙规则会被清空（重启后重新添加即可）
- MDM 注册信息删除：公司电脑可能无法访问内网资源（个人电脑无影响）

### 不受影响

- 不影响系统激活状态
- 不影响已安装的软件
- 不影响用户文件和数据
- 不影响网络连接能力
- 不影响 Windows Update 正常更新

## 常见问题

Q: 运行后提示"无法识别为 cmdlet"？
A: 右键 PowerShell → 以管理员身份运行，然后执行：
   Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
   再双击 bat 启动器即可。

Q: 重启后仍然显示"由组织管理"？
A: 再次运行脚本，诊断步骤会检查残留项。部分策略可能在重启后由系统重建，再次清理即可彻底清除。

Q: 这是公司电脑能用吗？
A: 可以运行，但清除后可能无法访问公司内网资源或公司配置的应用。建议先咨询 IT 管理员。
